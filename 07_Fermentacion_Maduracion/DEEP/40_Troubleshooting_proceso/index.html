    40 — Troubleshooting de Proceso (síntomas → gates → causas → acciones)¶

    Objetivo¶

    Resolver desvíos operativos de fermentación/maduración con un flujo consistente: 1) Confirmar datos 2) Evaluar gates 3) Diagnosticar causas probables (proceso) 4) Ejecutar acciones seguras 5) Registrar para prevenir recurrencia

    0) Triage rápido (5 minutos)¶

    Confirmar antes de tocar nada: - Temperatura real (sonda y ubicación) - Densidad (instrumento correcto + corrección por temperatura) - Tiempo desde inoculación - Si hubo cambios de setpoint o fallas del equipo - Si hubo transferencias / purgas / movimientos recientes - Nota sensorial rápida (manteca/solvente/frutado/sulfuro)

    1) “No arranca” o arranque muy lento¶

    Síntomas¶

    • 0–24h sin caída de densidad o sin señales.

    Gate¶

    • ¿Temperatura real está dentro del rango de la cepa?
    • ¿Hay overshoot (pico) o está demasiado fría?

    Causas (proceso)¶

    • Setpoint inicial mal elegido / control térmico deficiente
    • Mosto demasiado frío al inocular
    • Pérdida de control (sonda mal ubicada)

    Acciones (seguras)¶

    • Ajustar setpoint gradualmente al rango correcto.
    • No hacer cambios bruscos.
    • Registrar el perfil térmico real para corregir capacidad de enfriamiento.

    2) Overshoot térmico (pico de temperatura en fase activa)¶

    Síntomas¶

    • Temperatura real supera setpoint en 24–72h iniciales.

    Riesgo¶

    • Mayor probabilidad de perfil no deseado: ésteres/fuseles y riesgo de diacetilo si el cierre luego se corta.

    Acciones¶

    • Bajar setpoint de forma controlada (evitar “shock”).
    • Recalibrar ubicación/aislación de sonda.
    • Para próximos lotes: setpoint inicial más conservador + capacidad real de enfriamiento.

    3) Fermentación lenta (caída de densidad se “aplana”)¶

    Gate¶

    • ¿Temperatura se mantiene estable o fluctúa?
    • ¿Hay rampa final aplicada demasiado temprano o demasiado tarde?

    Causas (proceso)¶

    • Oscilación térmica / setpoint inadecuado
    • Enfriamiento accidental (ambiente, chiller, sonda)

    Acciones¶

    • Estabilizar temperatura dentro del rango.
    • Si está en fase final: rampa moderada de cierre (sin excederse).
    • Registrar el momento exacto donde se aplanó la curva.

    4) “Stuck” o FG alta (no llega al objetivo)¶

    Gate¶

    • ¿Densidad estable por 48h?
    • ¿Se aplicó cierre/maduración suficiente?

    Causas (proceso)¶

    • Cortaste temperatura demasiado pronto
    • Crash / transferencia temprano
    • Secuencia de rampas incorrecta

    Acciones¶

    • Subir temperatura gradualmente hacia el extremo alto del rango de cepa (cierre).
    • Evitar transferencias hasta estabilizar.
    • Documentar: densidad, temperatura real y tiempos.

    5) Gate de diacetilo falla (manteca / precursor)¶

    Síntomas¶

    • Manteca perceptible o test de diacetilo positivo.

    Causas (proceso)¶

    • Enfriaste o separaste de levadura demasiado pronto
    • No hubo descanso/rampa final

    Acciones¶

    • Volver a condiciones de maduración (temperatura de cierre) y esperar limpieza.
    • No cold crash / no transferir hasta pasar gate.

    6) Después de transferir aparece “papel/cartón” o envejecimiento acelerado¶

    Síntomas¶

    • Perfil oxidado, decaimiento rápido, apagado aromático.

    Causas (proceso)¶

    • Ingreso de oxígeno en transferencia (mangueras con aire, salpicado, headspace, purga insuficiente)

    Acciones¶

    • Tratarlo como evento de proceso: revisar purgas, procedimiento de llenado, headspace.
    • Establecer gate de “transferencia controlada” (checklist obligatorio).

    7) Cold crash no clarifica como se espera¶

    Gate¶

    • ¿Estaba realmente cerrado el lote antes de crash?
    • ¿Tiempo/temperatura de crash fueron suficientes?

    Causas (proceso)¶

    • Crash demasiado corto o no lo suficientemente frío
    • Lote no cerrado (azúcar residual / fermentación incompleta)
    • Objetivo de claridad no definido por estilo

    Acciones¶

    • Aumentar tiempo en frío (con registro).
    • Confirmar densidad estable antes del crash.
    • Si aplica al estilo/proceso: plan de clarificación (ver puente a 03_Levadura/Bloque 05).

    8) Checklist de prevención (sistema)¶

    • [ ] Registrar temperatura real (no solo setpoint)
    • [ ] Registrar densidad (serie)
    • [ ] Definir gates: inicio / fase final / cierre / frío / salida a empaque
    • [ ] No transferir ni crash antes del gate de cierre
    • [ ] Transferencias con control de oxígeno (procedimiento y checklist)
    • [ ] Cold crash con objetivo claro (claridad/estabilidad), y tiempo suficiente

    Puentes (no duplicar)¶

    • Pitch rate, oxígeno, repitch, floculación: ver 03_Levadura (Fermentación_DEEP).
    • Empaque/estabilidad (DO en envase, purgas específicas, shelf-life): ver 09_Empaque_Estabilidad.
