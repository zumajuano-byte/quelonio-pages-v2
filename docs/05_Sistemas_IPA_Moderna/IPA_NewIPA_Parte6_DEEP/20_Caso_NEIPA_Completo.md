# Caso NEIPA completo (sistema aplicado)

## 1) Objetivo sensorial (ejemplo)
- “Tropical alto, amargor suave, mouthfeel soft/juicy, haze estable, cero oxidación perceptible.”

## 2) Arquitectura recomendada
### Amargor (capa A)
- Bajo a medio-bajo, diseñado para “estructura” sin dureza.
- Evitar depender de IBU teórico: priorizar calidad.

### Hot-side (capa B)
- Whirlpool con temperatura/tiempo controlados para maximizar aroma sin castigo vegetal.

### Cold-side (capa C)
- Dry hop principal post-fermentación (retención + control).
- Etapa temprana opcional si buscás biotransformación, con plan anti-hop creep.

Puentes:
- Parte 2: `../IPA_NewIPA_Parte2_DEEP/00_INDEX.md`
- Parte 3: `../IPA_NewIPA_Parte3_DEEP/00_INDEX.md`

## 3) Receta como plataforma (principios)
- Base limpia.
- Construcción de textura sin empalagar: mash + soporte, no “azúcar residual”.
- Haze como matriz controlada (proteína + polifenol + proceso).

Puente:
- Parte 5 (haze/hop burn/shelf): `../IPA_NewIPA_Parte5_DEEP/00_INDEX.md`

## 4) Riesgos típicos y controles
- Hop burn / astringencia: limitar contacto, controlar temperatura, mejorar separación.
- Oxidación: proceso cerrado + purgas + estándar de transfer/envasado.
- Hop creep/diacetilo: gate de VDK y planificación del timing.
