# 20 — Cómo pasar sensorial a Spec y Receta (método simple)

## Objetivo (en simple)
Convertir lo que “sentimos” en cata en cambios concretos y controlados:
- del **Spec** (qué debe ser la cerveza),
- y/o de la **Receta/Proceso** (cómo lograrlo),
sin improvisar y sin cambiar 10 cosas juntas.

---

## 1) Regla de oro: separar 3 tipos de problema
Cuando algo no cierra en cata, primero clasificar:

### A) Problema de proceso (no de receta)
Ejemplos: oxidación, diacetilo, contaminación, “apagado” por mala guarda.  
Acción: corregir SOP (07/09/10) antes de tocar receta.

Puentes:
- Oxígeno/DO/TPO: [09/20 — Control de oxígeno](../../09_Empaque_Estabilidad/DEEP/20_Control_oxigeno_DO_TPO.md)
- Almacenamiento: [09/30 — Luz/temperatura](../../09_Empaque_Estabilidad/DEEP/30_Luz_temperatura_almacenamiento.md)
- Troubleshooting: [09/40 — Shelflife](../../09_Empaque_Estabilidad/DEEP/40_Shelflife_y_troubleshooting_empaque.md)
- Limpieza/CIP: [10/20 — Troubleshooting CIP (Kunze)](../../10_Limpieza_Sanitizacion/DEEP/20_Troubleshooting_CIP_Kunze_Nivel2.md)

### B) Problema de spec (el objetivo estaba mal definido)
Ejemplo: “para este estilo queremos más seco/menos amargo/más aroma”, pero nunca lo dejamos escrito.  
Acción: ajustar Spec (objetivos y tolerancias), no receta todavía.

### C) Problema de receta (el objetivo es correcto, pero no llegamos)
Ejemplo: el spec pide cierto perfil, pero la receta no lo consigue.  
Acción: cambiar receta/proceso, pero con método (ver sección 3).

---

## 2) Formato de “ticket” de mejora (1 por lote)
> Siempre nace del TP sensorial.

### Inputs obligatorios
- Lote:
- Estilo / Producto:
- Fecha de cata:
- Decisión: OK / Dudoso / No OK
- “Sospecha” (1 frase):
- ¿Es A, B o C? (proceso / spec / receta)

➡️ Fuente: [TP — Log de Cata](TP_Log_Cata_y_Defectos.md)

---

## 3) Método de cambio: “una variable por corrida”
Para evitar caos:
1) Elegí **1 variable principal** a cambiar.
2) Definí la **hipótesis** (qué esperás que cambie y por qué).
3) Definí cómo vas a medir si funcionó (sensory + 1 dato simple si aplica).
4) Guardalo en registro.

Ejemplos de variables (no mezclar muchas):
- receta: % malta especial / tipo de lúpulo / carga total / timing
- proceso: temperatura de maceración / curva térmica / tiempo de hervor / manejo de oxígeno / temperatura de guarda
- empaque: purga, headspace, tiempo entre llenado y cierre

---

## 4) Traducción rápida: síntoma → dónde tocar
### “Aroma apagado” (especialmente hoppy)
Primero: proceso/empaque (oxígeno + temperatura).  
Recién después: receta de lúpulo.

### “Manteca / diacetilo”
Proceso (07). No se corrige con receta.

### “Astringencia / áspero”
Puede ser proceso (maceración, lavado, pH) o receta (cáscara, cargas).  
Acción: elegir 1 hipótesis y probar.

### “Demasiado dulce / poco seco”
Proceso (fermentabilidad / atenuación) o receta (maltas).  
Acción: 1 ajuste por vez (ej. temperatura de maceración o levadura).

---

## 5) Actualizar Spec vs actualizar Receta (cuándo corresponde)
### Actualizar Spec cuando:
- cambió la visión del producto (“queremos más seco / más amargo / más aroma”)
- el mercado/objetivo cambió
- el spec era vago (no tenía tolerancias)

### Actualizar Receta cuando:
- el spec es correcto, pero el lote no lo alcanzó
- repetís el mismo desvío en 2+ lotes

---

## 6) Salidas (qué queda escrito)
Por cada lote con mejora:
- TP sensorial (ya existe)
- “Ticket de mejora” (este doc define el formato)
- cambio aplicado (receta o SOP)
- resultado (cata a 7/14/30 días)
