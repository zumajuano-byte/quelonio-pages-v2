# 03_Auditorias_QMS

## Por qué auditar
Auditar no es “pasar lista”: sirve para **detectar brechas**, forzar alineación de equipos y mantener el sistema vivo.

## Estructura recomendada (3 partes)
Dividir una auditoría de sistema en:
- **Process assessment** (procesos)
- **Department assessment** (política y personas)
- **Product assessment** (producto)
Esto ayuda a enfocar y acotar el esfuerzo. fileciteturn23file6L9-L22

## Resultados típicos (clasificación simple)
Para cada proceso auditado, clasificar:
- SOPs existen y se siguen (sin brechas)
- SOPs mayormente, con brechas de consistencia
- Falta significativa de SOPs o inconsistencia entre staff fileciteturn23file6L26-L30

## Qué revisar (extracto operativo)
Ejemplos de ítems de process assessment:
- **Lab SOPs**: todos los tests con procedimiento escrito + consistencia entre operadores. fileciteturn23file6L46-L50
- **Taste panel**: registros de entrenamiento, criterios claros de hold/release, consistencia entre catadores. fileciteturn23file6L51-L53
- **Brewing QC**: pH, Plato, conteos, etc. con SOP y seguimiento. fileciteturn23file6L54-L58
- **Packaging QC**: DO, CO₂, costuras, inspección visual; SOP + adherencia. fileciteturn23file4L1-L7
- **Utilities QC**: calidad de agua/steam/CO₂ con SOP. fileciteturn23file4L17-L23
- **QA metrología**: mantenimiento/calibración de instrumentos, inline/offline, trazabilidad. fileciteturn23file4L34-L43

## Cadencia sugerida (simple)
- Trimestral: packaging QC + metrología
- Semestral: brewhouse QC + utilities
- Anual: auditoría de sistema (3 partes) + 3rd party si aplica
- 6 meses post auditoría: follow-up a correcciones. fileciteturn23file6L39-L41
