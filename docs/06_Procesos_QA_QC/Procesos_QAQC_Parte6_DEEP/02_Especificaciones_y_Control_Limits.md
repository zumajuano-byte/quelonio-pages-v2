# 02_Especificaciones_y_Control_Limits

## Definiciones operativas
- **Especificaciones**: límites cuantificables de parámetros de proceso o producto basados en requerimientos del cliente. fileciteturn23file1L14-L16
- **Control limits**: límites estadísticos (idealmente dentro de specs) que requieren reacción cuando el proceso/producto *tiende* a exceder una especificación. fileciteturn23file1L16-L19

## Reglas prácticas (para no sobrerreaccionar)
1) Setear specs por requerimiento del usuario final; control limits con control charts. fileciteturn23file1L31-L35
2) Priorizar riesgos y parámetros clave. fileciteturn23file1L35-L36
3) Definir exactamente qué hacer ante breach, evitando reacción costosa. fileciteturn23file1L37-L39

## Traducción a cerveza (ejemplo conceptual)
- Si el consumidor no distingue BU 30 vs 36, no poner la SPEC más baja que el umbral sensorial (y sostener control limits más finos internamente). fileciteturn23file1L55-L60

## Checklist para crear una SPEC (por parámetro)
- ¿Quién es el cliente (consumidor / bar / distribuidor / interno)?
- ¿Qué defecto percibe y en qué umbral?
- ¿Qué variable lo predice? (DO, temp, pH, CO₂, etc.)
- ¿Qué variable es más fácil de controlar en planta?
- ¿Se puede medir de forma repetible? ¿Está calibrado el equipo?
