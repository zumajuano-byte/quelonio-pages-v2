# Política — Biblia (v2) + Fuentes (Drive) + RAG (soporte)

## 1) Principio
- La **Biblia v2** es el **cerebro operacional** (SOPs, reglas, checklists, estándares).
- Los **PDFs** viven **privados en Google Drive** y se usan como **soporte amplificado** (evidencia y bibliografía).
- El repo v2 **no** contiene PDFs ni carpetas bibliográficas duplicadas.

## 2) Regla de oro (validez de reglas)
Una regla solo es válida si cita:
- `FuenteID`
- `Ubicación exacta` (p/fig/tbl)
- `Cita textual corta` (<= 25 palabras)

## 3) Registro de fuentes
El catálogo canónico de fuentes es:
- `99_Indice_y_Mapas/REGISTRO_FUENTES.md`

## 4) Cómo se usa la RAG en este proyecto
- RAG sirve para **encontrar y recuperar evidencia** desde PDFs (Drive).
- RAG puede proponer fragmentos y síntesis, pero **no decide el estándar**.
- El estándar final se fija y mantiene en la Biblia v2.

## 5) Publicación
- v2 publica solo contenido canónico (Markdown).
- Los PDFs permanecen privados; se publican metadatos (FuenteID) y citas cortas.
