# Checklist — Post-fermentación (V2)

Este checklist cierra el lote y define si la levadura es apta para repitch.
(Parte 4: post-fermentación, cosecha, reutilización y estabilidad)

---

## 1) Limpieza de fermentación
- [ ] DF estable (confirmado)
- [ ] Diacetilo reducido (sin notas VDK)
- [ ] Perfil sensorial “limpio” vs objetivo (sin defectos dominantes)

<a id="r-0117"></a>
### R-0117 — No cerrar sin limpieza confirmada
- FuenteID: Q-STD-LEVADURA (Checklist Parte 4)
- Evidencia: "Diacetilo reducido"
- Ubicación: Parte 4 → Checklist → Limpieza de fermentación

Regla (operativa):
- No ejecutar decisiones irreversibles (crash/transfer/cosecha) sin confirmar limpieza (VDK/diacetilo según tu práctica).

---

## 2) Floculación / sedimentos
- [ ] Floculación completa (según cepa/estilo)
- [ ] Sedimentos coherentes (sin señales extrañas)
- [ ] Si hubo clarificación: decisión registrada (qué y por qué)

<a id="r-0118"></a>
### R-0118 — Floculación completa antes de acciones finales
- FuenteID: Q-STD-LEVADURA (Checklist Parte 4)
- Evidencia: "Floculación completa"
- Ubicación: Parte 4 → Checklist → Floculación/sedimentos

Regla (operativa):
- Evitar “cortar” el lote por apariencia. Si la levadura cae temprano y el lote no está limpio, aumenta el riesgo de defectos y DF fuera de target.

---

## 3) Cosecha (si aplica)
- [ ] Cosecha limpia (sin arrastre evidente)
- [ ] Ventana óptima respetada (cosecha en momento planificado)
- [ ] Minimizar oxígeno durante la cosecha (manejo cerrado)

<a id="r-0119"></a>
### R-0119 — Cosecha limpia y en ventana óptima
- FuenteID: Q-STD-LEVADURA (Parte 4)
- Evidencia: "Cosechar en ventana óptima" + "Cosecha limpia"
- Ubicación: Parte 4 → Cosecha y reutilización / Checklist

Regla (operativa):
- Cosechar solo si se cumple: ventana óptima + manejo limpio + oxígeno minimizado.

---

## 4) Viabilidad / vitalidad (criterio mínimo)
- [ ] Viabilidad evaluada (método o criterio definido)
- [ ] Vitalidad aceptable (criterio definido)
- [ ] Sin caída evidente de performance vs lotes anteriores (si hay histórico)

<a id="r-0120"></a>
### R-0120 — Si no medís, registrá criterio (viabilidad/vitalidad)
- FuenteID: Q-STD-LEVADURA (Checklist Parte 4)
- Evidencia: "Viabilidad evaluada"
- Ubicación: Parte 4 → Checklist → Viabilidad/vitalidad

Regla (operativa):
- Prohibido “asumir” viabilidad. Si no hay medición, dejar criterio explícito y repetible.

---

## 5) Repitch / reutilización
- [ ] Generación registrada
- [ ] Condición de almacenamiento definida (frío/limpio)
- [ ] Plan de repitch definido (cuándo, cuánto, cómo)

<a id="r-0121"></a>
### R-0121 — Generación registrada es obligatoria
- FuenteID: Q-STD-LEVADURA (Checklist Parte 4)
- Evidencia: "Generación registrada"
- Ubicación: Parte 4 → Checklist → Repitch/reutilización

Regla (operativa):
- No reutilizar sin registrar generación y “historia” mínima del cultivo.

<a id="r-0122"></a>
### R-0122 — Almacenamiento frío y limpio (guardrail)
- FuenteID: Q-STD-LEVADURA (Parte 4)
- Evidencia: "Almacenar frío y limpio"
- Ubicación: Parte 4 → Cosecha y reutilización

Regla (operativa):
- Si vas a repitear, el estándar mínimo es almacenar frío y limpio; cualquier desvío exige justificar o descartar.

---

## 6) Decisión final (GO / NO-GO)
- [ ] GO repitch (cumple limpieza + cosecha + viabilidad + generación + almacenamiento)
- [ ] NO-GO repitch (descartar o “reset” de cepa) — causa registrada

<a id="r-0123"></a>
### R-0123 — Reutilizar levadura es gestión biológica (no “atajo”)
- FuenteID: Q-STD-LEVADURA (Parte 4)
- Evidencia: "Reutilizar levadura es gestión biológica."
- Ubicación: Parte 4 → Cosecha y reutilización

Regla (operativa):
- La decisión de reutilizar se toma con criterios de riesgo; no por conveniencia.

<a id="r-0124"></a>
### R-0124 — Siempre dejar trazabilidad final del lote
- FuenteID: Q-STD-LEVADURA (Checklist Parte 4)
- Evidencia: "Generación registrada" + "Registro recomendado"
- Ubicación: Parte 4 → Checklist → Registro

Regla (operativa):
- Si no queda registro final, el lote no “enseña” y no se puede repetir ni diagnosticar.
